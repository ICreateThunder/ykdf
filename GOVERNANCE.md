# Governance

## Current State

Single maintainer: [@ICreateThunder](https://github.com/ICreateThunder)

This is a practical constraint, not a preference. The project will transition to
multi-maintainer governance when external contribution demonstrates traction.

## Roles

**Contributors** - anyone with a merged PR, issue, or discussion participation.

**Maintainers** - can review/merge PRs, triage issues, manage releases.
Commit to 7-day PR review SLA and security response SLAs per SECURITY.md.

## Decisions

- **Day-to-day:** code review + maintainer approval + green CI = merge
- **Architectural:** requires discussion in an issue with a minimum 7-day comment period
- **Disputes:** maintainer decides with written justification

## Transition Plan

When 3+ active maintainers exist beyond the original:
- Consensus for normal changes
- Lazy consensus (7 days) for non-controversial changes
- Simple majority vote for contested decisions
